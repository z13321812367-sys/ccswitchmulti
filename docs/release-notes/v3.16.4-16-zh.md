# CCSwitchMulti v3.16.4-16

v3.16.4-16 是 `v3.16.4-15` 之后的 Codex MultiRouter、模型菜单解锁和子 Agent 兼容修复版本。重点修复新版 Codex Desktop/CLI 下自定义模型显示、托管子 Agent 角色同步，以及 Qwen/vLLM 等 Chat 上游的 Responses 转换语义。

## 新增与改进

- Codex Desktop 模型菜单解锁入口增加明确提示：需要完全退出普通启动的 Codex Desktop 后再由 CCSwitchMulti 启动或连接带 remote debugging 的实例；按钮 hover 会说明它只注入 renderer 模型白名单，不会改 API Key、路由或模型目录。
- 模型菜单解锁现在会在接管开启或确认已接管后 best-effort 自动尝试一次；如果现有 Desktop 没有 CDP 端口，仍保留用户显式点击按钮的安全路径，避免静默重启用户窗口。
- CCSwitchMulti 托管的 Codex 子 Agent role 会清理不再属于当前候选窗口的历史角色，避免新版 Codex 智能体列表继续显示旧自定义模型。
- Qwen 托管子 Agent role 不再硬编码 `model_reasoning_effort=low`，让新版 Codex 子 Agent 继承当前会话或模型默认 reasoning effort。
- release workflow 和下载说明统一恢复 `CCSwitchMulti-*` 外显资产命名，Windows portable 包内可执行文件也使用 `CCSwitchMulti.exe` 展示名。

## Bug 修复与护栏

- 修复 Codex Desktop portable/CLI 识别边界：Desktop 和 CLI 都支持，但进程过滤不再误排除小写 `codex.exe` 的 CLI/app-server 场景。
- 修复 API Key 登录或无官方 ChatGPT 登录态时的模型菜单说明，明确这是 Codex Desktop renderer 门控边界，不把它误判成 catalog 或路由丢失。
- 修复 Qwen/vLLM reasoning 配置被旧 DB meta 覆盖的问题：识别到 Qwen + vLLM/matrixminecraft 时，会把旧 `thinking` 参数兼容回 `enable_thinking`，并禁用不适用的 `reasoning_effort` 透传。
- 修复历史自动 `defaultOutputTokens=32768` 改变 Codex 缺省输出预算语义的问题。Codex 没有声明 `max_output_tokens/max_tokens/max_completion_tokens` 时，CCSwitchMulti 默认不再为 Qwen/vLLM 自动补 `max_tokens`；`minOutputTokens` 只抬高已经存在且过小的显式预算。
- 补强 Codex official 子 Agent token 统计与 MultiRouter 请求形态日志，便于排查 Responses -> Chat 转换后的 `max_tokens`、`enable_thinking`、`reasoning_effort` 实际出站字段。
