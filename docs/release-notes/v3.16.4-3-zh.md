# CCSwitchMulti v3.16.4-3

v3.16.4-3 是基于 v3.16.4-2 的正式发布版本，重点补齐 Codex MultiRouter 首次配置向导、混合中转 provider 的协议拆分、Responses 能力探测、子 Agent 用量统计和 Windows 图标安装链路。这个版本面向正在使用 Codex Desktop + 多模型路由的用户，目标是减少手动配置、避免协议误判，并让子 Agent 成本统计更可读。

## 重点更新

### Codex MultiRouter 配置向导

- 新增 Codex 入口里的 MultiRouter 创建向导，引导用户按步骤添加模型源、检查 API Key / Base URL / API 格式、获取模型列表、处理同名模型，并生成可发布的多模型路由配置。
- 向导改为显式状态机，覆盖缺模型源、配置不完整、获取模型中、部分失败、模型冲突确认、保存失败、启用失败、完成等状态，避免线性步骤在异常场景里跳错页。
- 向导保存仍然只在用户点击保存发布时写入 provider，不会在浏览或刷新模型时静默切换当前 Codex provider。
- 向导完成后可以直接启用生成的 MultiRouter provider，并复用现有本地接管、OAuth 保留和路由诊断链路。

### 中转 provider 混合协议拆分

- 在普通 Codex provider 表单点击“获取模型”时，如果发现同一个中转同时返回 GPT-like 模型和 Qwen / DeepSeek 等非 GPT-like 模型，会弹出确认提示。
- 确认后不再向同一个 provider 内写两条隐藏 route，而是在新增保存时生成两个独立 provider：
  - `<provider>-responses`：走 `openai_responses`，只保留 GPT / Responses 候选模型目录。
  - `<provider>-chat`：走 `openai_chat`，只保留 Chat Completions 候选模型目录。
- 这样 provider 列表里能直接看到两种协议来源，避免后续编辑时分不清“同一个中转里哪部分走 Responses、哪部分走 Chat 转换”。
- 编辑已有 provider 时不会自动生成额外 provider，避免“编辑 A 生成 B/C”的高风险行为。

### Responses 能力探测

- 新增显式的 Codex Responses 探测能力，向导可以对候选 provider 发起最小 `/v1/responses` 探测，并把 HTTP 状态、目标 URL、模型和错误摘要结构化展示出来。
- 探测不会静默修改配置，也不会缓存成永久能力结论；它只作为配置前的可见诊断，避免把临时网络错误、账号权限问题或上游限流误判成模型协议能力。
- URL 归一化支持 provider 根地址、`/v1`、完整 `/v1/chat/completions` 等输入，统一收敛到 `/v1/responses` 探测路径。

### Responses-Lite fallback 更稳

- Codex 私有 `x-openai-internal-codex-responses-lite` 头改为更保守的 fallback 处理：默认按原请求发送；当上游明确返回 Lite 不支持错误时，剥离该 header 重试一次。
- 对失败过的 provider / URL / 模型组合加入短期负缓存，缓存期内直接去掉 Lite header，避免每次请求都先失败一次。
- fallback 缓存设置过期时间，未来上游支持 Lite 后仍有机会重新探测，不会永久禁用。

### 子 Agent 用量统计按模型聚合

- 修复子 Agent 今日流量统计全 0 的问题：同步 Codex 子 Agent 用量时改用 rollout 线程 ID，避免把子 Agent 请求错误归到父线程 ID。
- 子 Agent 用量视图改为按模型聚合，展示每个模型的子 Agent 数量和用量，而不是逐个列出大量子 Agent 会话。
- 即使某个子 Agent 暂时没有用量，也会参与模型维度的 Agent 数统计，便于判断模型分配情况。

### Windows 图标和安装体验

- 刷新 CCSwitchMulti 应用图标，并重新生成 Tauri / Windows / iOS 尺寸图标资源。
- Windows NSIS 安装包显式使用新图标，并在安装后重写开始菜单和桌面快捷方式的 `IconLocation`，减少重新安装后任务栏或快捷方式仍显示旧图标的情况。
- 新增 Windows 安装图标验证脚本，便于检查源 ico、安装目录 exe 内嵌图标和快捷方式图标目标是否一致。

### 子 Agent 派生体验

- MultiRouter 投影 Codex 模型目录时保持 `spawn_agent` 的模型覆盖信息可见，避免 Codex Desktop 派生子 Agent 时看不到 MultiRouter 提供的候选模型。
- 如果已有 `features.multi_agent_v2` 配置，会保留原有启用状态，只补充 `hide_spawn_agent_metadata=false`，不强行改变用户原有子 Agent 配置语义。

## 已知边界

- Windows 本地构建直接产出 Windows 安装包、便携包、raw exe 和 updater `latest.json`。Linux / macOS 二进制仍需要对应平台或补充工作流构建。
- Responses 能力探测是显式诊断，不是永久能力缓存；如果上游策略、账号权限或模型白名单变化，需要重新探测。
- 如果 Windows 任务栏固定项仍显示旧图标，通常是 Explorer / 任务栏固定项缓存，需要刷新固定项或重启 Explorer；安装包和快捷方式图标已按新图标写入。
